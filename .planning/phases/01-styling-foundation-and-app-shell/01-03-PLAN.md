  <action>
The core problem: glass.css defines hardcoded rgba() panel backgrounds per-theme that all look nearly identical. The original theme.css already has rich per-theme tokens (--surface-1, --surface-2, --border, --cyan, --grad-space, --shadow-card) that give each theme its unique feel. We need to use those instead.

**STEP 1 — Gut glass.css, keep only sidebar tokens:**

Reduce glass.css to sidebar-only. Remove all `--glass-panel-*` and `--glass-card-*` tokens. The sidebar can keep its glass treatment (it's the anchor), but reduce blur from 12px to 6px for subtlety. Keep per-theme sidebar overrides.

```css
/* glass.css — sidebar glass only. Content panels use theme tokens directly. */
:root {
  --glass-sidebar-bg: rgba(15, 23, 42, 0.88);
  --glass-sidebar-blur: 6px;
  --glass-sidebar-border: rgba(59, 130, 246, 0.12);
}
/* Keep existing per-theme sidebar overrides, remove everything else */
```

**STEP 2 — Rewrite GlassPanel to use theme tokens:**

Replace inline glass styles with Tailwind theme classes:

```tsx
// Non-classic (glass themes):
// - Background: bg-theme-surface1/60 (semi-transparent, theme-colored)
// - Border: border border-theme-border/30, with optional accent border via prop
// - Shadow: shadow-[var(--shadow-card)]
// - NO backdrop-filter, NO blur
//
// Classic (Mario):
// - Keep existing solid retro panel (unchanged)
```

The key change: `bg-theme-surface1/60` uses the theme's own surface color at 60% opacity. For Hyperborea that's a blue-tinted panel, for Synthwave it's a purple-tinted panel, for Tropical it's teal-tinted. Each theme looks different.

Add an optional `accent` prop to GlassPanel for colored border highlights:
- `accent="cyan"` → `border-theme-cyan/40` (like the selected card in the screenshot)
- `accent="warning"` → `border-theme-warning/40` (like the gold-bordered card)
- Default: `border-theme-border/30`

**STEP 3 — Rewrite GlassCard to use theme tokens:**

Same approach for inner cards:
- Background: `bg-theme-surface2/50`
- Border: `border border-theme-border/20`
- Hover: `hover:bg-theme-surface2/65`
- NO blur

**STEP 4 — Update AppShell sidebar styling:**

The sidebar `<aside>` currently uses inline styles referencing glass tokens. Update to use the remaining sidebar glass tokens (which are still in glass.css), but with reduced blur.

**STEP 5 — Typography hierarchy (same as before, plus accent colors):**

Keep the typography utilities already added to app.css (typo-h1 through typo-value). These are already correct from the previous 01-03 execution attempt.

Add accent-colored section header variant:
```css
.typo-section { @apply text-xs font-bold uppercase tracking-[0.2em] text-theme-cyan; }
```
This matches the screenshot where section titles like "BULL SCENARIO", "MODEL STACK" use the theme's cyan accent color in uppercase tracking.

**STEP 6 — Spacing standardization:**

Same compact spacing grid as before — audit all Plan 01/02 components. The spacing comments are already in app.css from the previous attempt.

**STEP 7 — Hover and focus-visible states:**

Same interaction states as before, but using theme tokens not glass tokens:
- Sidebar nav items: `hover:bg-theme-cyan/10`
- GlassPanel hover: `hover:bg-theme-surface1/70` (brighten slightly)
- GlassCard hover: `hover:bg-theme-surface2/65`
- Focus-visible: `focus-visible:outline-2 focus-visible:outline-theme-cyan`

**STEP 8 — Verify canvas visibility:**

- AppShell content area: bg-transparent (no solid bg)
- GlassPanel: 60% surface opacity → background shows through
- GlassCard: 50% surface-2 opacity → background shows through
- Sidebar: 88% glass opacity (solid anchor, acceptable)
- `--grad-space` gradient from theme.css provides the ambient color wash behind everything
