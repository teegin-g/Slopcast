  <action>
These components use inline `style={{}}` with hardcoded hex/rgba colors that cannot be replaced with Tailwind classes because they apply to canvas elements, SVG fills, or dynamic positioning.

**WaterfallChart.tsx:**
The chart uses hardcoded hex colors like `#3b82f6` (blue), `#E566DA` (pink) for bar fills. Replace with CSS custom property references:
- Read the component to understand how colors are applied (likely `style={{ backgroundColor: '#3b82f6' }}` or canvas fillStyle)
- If using inline styles on DOM elements: replace with `style={{ backgroundColor: 'var(--cyan)' }}` or `'var(--magenta)'` etc.
- If using canvas API (fillStyle, strokeStyle): use `getComputedStyle(document.documentElement).getPropertyValue('--cyan').trim()` to read theme colors at render time
- Map existing colors to theme tokens: blue -> `--cyan`, pink/magenta -> `--magenta`, green -> `--success`, red -> `--danger`, yellow/gold -> `--warning`

**MiniMapPreview.tsx:**
Has inline `rgba()` for grid lines, fills, and strokes. Replace:
- Grid line colors: `rgba(255,255,255,0.05)` -> `var(--border)` with opacity applied via canvas globalAlpha or rgba decomposition
- Fill colors: `rgba(0,0,0,0.3)` -> use `var(--surface-1)`
- If canvas API: read CSS custom properties via getComputedStyle at component mount/theme change
- If inline styles on DOM elements: use `var(--border)`, `var(--surface-1)` directly

**For canvas-based rendering**, create a helper at the top of the component:
```typescript
function getThemeColor(prop: string): string {
  return getComputedStyle(document.documentElement).getPropertyValue(prop).trim();
}
```
Call this inside the draw/render function so it picks up theme changes.
