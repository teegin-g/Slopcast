  <behavior>
    InlineEditableValue:
    - Renders formatted display value as a span at rest
    - Click on span enters edit mode (shows input with current value selected)
    - onFocus on span also enters edit mode (for Tab navigation)
    - Hover shows bg-theme-surface2/50 highlight + cursor-pointer (hover affordance only, per user decision)
    - Blur calls onCommit with parsed value, exits edit mode
    - Enter key calls onCommit with parsed value, exits edit mode
    - Escape reverts draft to original value, exits edit mode, does NOT call onCommit
    - If validate returns error string: red border on input, tooltip below with error, blur does NOT commit
    - Accepts format prop for display formatting (raw number during edit per Claude discretion -- avoid cursor jump)
    - Accepts parse prop to transform string input back to value type
    - Accepts type prop ('text' | 'number', default 'text')
    - Accepts className and inputClassName for theme branching (isClassic support)

    useDebouncedRecalc:
    - Accepts onUpdateGroup callback and delay (default 400ms)
    - Returns { debouncedUpdate, isRecalculating }
    - Calling debouncedUpdate multiple times within delay window: only last group is passed to onUpdateGroup
    - Uses ref (latestGroupRef) to avoid stale closure pitfall
    - isRecalculating is true from first call until ~150ms after recalc fires
    - Cleans up timer on unmount
  </behavior>
  <action>
    Create `src/components/inline/` directory.

    **InlineEditableValue.tsx** (~80-100 lines):
    Follow the pattern from 03-RESEARCH.md Pattern 1 closely. Key details:
    - Props interface: value, onCommit, format?, parse?, validate?, type?, className?, inputClassName?
    - Internal state: editing (boolean), draft (string), error (string | null)
    - inputRef for focus management
    - useEffect on editing change: when entering edit mode, set draft to String(value), clear error, requestAnimationFrame to select input
    - commit(): validate -> if error, set error and return -> parse -> onCommit -> setEditing(false)
    - cancel(): revert draft, clear error, setEditing(false)
    - handleKeyDown: Enter -> commit, Escape -> cancel
    - Display mode: span with onClick, onFocus, tabIndex=0, hover classes
    - Edit mode: relative wrapper with input + conditional error tooltip
    - Input classes: bg-theme-bg border rounded-inner px-1 text-theme-text, conditional red border on error, focus ring
    - Error tooltip: absolute positioned below input, text-[9px], red-400, z-50

    **InlineEditableValue.test.tsx**:
    Use Vitest + React Testing Library. Tests:
    1. Renders formatted display value
    2. Clicking enters edit mode with input
    3. Blur commits value via onCommit
    4. Enter commits value
    5. Escape cancels without calling onCommit
    6. Validation error prevents commit, shows error message
    7. Parse function transforms value before commit

    **useDebouncedRecalc.ts** (~30 lines):
    Follow 03-RESEARCH.md Pattern 2. Key details:
    - latestGroupRef stores most recent WellGroup
    - timerRef stores setTimeout handle
    - debouncedUpdate: store group in ref, set isRecalculating true, clear existing timer, set new timer
    - On timer fire: call onUpdateGroup with latestGroupRef.current, then setTimeout 150ms to set isRecalculating false
    - useEffect cleanup clears timerRef
    - useCallback wraps debouncedUpdate with [onUpdateGroup, delay] deps

    **useDebouncedRecalc.test.ts**:
    Use Vitest with vi.useFakeTimers(). Tests:
    1. Fires onUpdateGroup after delay with latest group
    2. Multiple calls within window: only last group sent to onUpdateGroup
    3. isRecalculating is true during debounce window
    4. isRecalculating becomes false after recalc + 150ms settle
    5. Cleanup on unmount clears pending timers
